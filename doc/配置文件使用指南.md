## 快速开始

### 第一次使用？按照这个步骤

#### 步骤1：选择配置模板

```bash
# 基础测试（推荐新手）
cp config.basic.json config.json

# 或者选择其他模板
# cp config.thinking_modes.json config.json
# cp config.scenarios.json config.json
# cp config.comprehensive.json config.json
# cp config.streaming.json config.json
# cp config.quick.json config.json
```

#### 步骤2：替换占位符

打开`config.json`，替换以下占位符：

| 占位符 | 替换为 | 示例 |
|--------|--------|------|
| `YOUR_BASE_URL` | 你的网关地址 | `https://your-gateway.example.com` |
| `YOUR_API_KEY` | 你的API密钥 | `sk-your-api-key-here` |
| `YOUR_CHAT_MODEL` | 你的聊天模型路径 | `your-model-name` |
| `YOUR_RESPONSES_MODEL` | 你的Responses模型路径 | `your-model-name` |
| `YOUR_EMBEDDING_MODEL` | 你的Embedding模型路径 | `your-embedding-model` |

**替换后示例**：
```json
{
  "base_url": "https://your-gateway.example.com",
  "api_key": "sk-your-api-key-here",
  "chat_model": "your-model-name",
  "responses_model": "your-model-name",
  "embedding_model": "your-embedding-model"
}
```

#### 步骤3：运行测试

```bash
# 使用run.sh脚本
./run.sh

# 或者直接使用locust
locust -f locustfile.py --users 10 --run-time 1m
```

## 配置模板说明

### 模板文件清单

项目提供了6个预配置的模板文件，按测试场景分类：

| 文件名 | 说明 | 主要用途 | 适用时机 |
|--------|------|---------|---------|
| **config.basic.json** | 基础测试 | 日常性能监控 | 日常监控、回归测试 |
| **config.thinking_modes.json** | 思考模式对比 | 不同推理等级对比 | 验证思考模式影响 |
| **config.scenarios.json** | 场景化测试 | 场景维度分析 | 分析场景性能差异 |
| **config.comprehensive.json** | 全面测试 | 综合性能验证 | 上线前全面测试 |
| **config.quick.json** | 快速验证 | 快速功能验证 | 开发测试、快速回归 |
| **config.streaming.json** | 流式接口测试 | 专注流式性能 | 测试流式接口 |

## 配置详解

### 1. 基础测试配置（config.basic.json）

**特点**：只包含常规接口，不启用思考模式和场景化测试

**适用场景**：
- 日常性能监控
- 功能回归测试
- 快速性能基准验证

**流量分配**：
- chat_stream: 50%
- chat_non_stream: 20%
- responses: 20%
- embeddings: 10%

### 2. 思考模式对比配置（config.thinking_modes.json）

**特点**：对比不同推理等级（off/low/medium/high）的性能差异

**适用场景**：
- 验证思考模型性能
- 对比不同推理等级差异
- 确定最佳推理配置

**流量分配**：
- chat_stream: 33%
- chat_non_stream: 17%
- responses_thinking_off: 12.5%
- responses_thinking_low: 12.5%
- responses_thinking_medium: 12.5%
- responses_thinking_high: 12.5%

**关键指标**：
- 思考时长（thinking_duration_ms）
- 输出TTFT（output_ttft_ms）
- 思考时间占比（thinking_ratio）

### 3. 场景化测试配置（config.scenarios.json）

**特点**：测试上下文长度和问题复杂度的组合影响

**适用场景**：
- 分析输入特征对性能的影响
- 为分级路由提供数据支持
- 容量规划和性能优化

**场景组合**：
- 短上下文 + 简单问题（scenario_short_simple）
- 短上下文 + 中等问题（scenario_short_moderate）
- 中等上下文 + 简单问题（scenario_medium_simple）
- 中等上下文 + 中等问题（scenario_medium_moderate）
- 中等上下文 + 复杂问题（scenario_medium_complex）
- 长上下文 + 中等问题（scenario_long_moderate）
- 长上下文 + 复杂问题（scenario_long_complex）

**关键指标**：
- 按上下文长度分组的性能数据
- 按复杂度分组的性能数据
- 场景影响系数分析

### 4. 全面测试配置（config.comprehensive.json）

**特点**：包含所有测试场景的综合配置

**适用场景**：
- 上线前全面性能验证
- 容量规划
- 性能基准建立

**注意**：
- 测试时间较长
- 建议在非业务高峰期执行
- 可根据实际需求调整权重

### 5. 快速验证配置（config.quick.json）

**特点**：轻量级配置，只测试核心接口

**适用场景**：
- 开发过程中的功能验证
- 快速回归测试
- 基本连通性测试

**测试接口**：
- chat_stream: 60%
- chat_non_stream: 20%
- responses: 20%

### 6. 流式接口测试配置（config.streaming.json）

**特点**：专注流式输出的性能测试

**适用场景**：
- 验证流式接口的TTFT、TPS等核心性能指标
- 测试思考模式的流式输出
- 分析流式场景的性能表现

## 自定义配置

### 修改流量权重

在 `traffic_ratio` 中调整各任务的权重：

```json
"traffic_ratio": {
  "chat_stream": 7,                      // 流式聊天权重
  "responses_thinking_medium": 3,        // 思考模式权重
  "scenario_medium_moderate": 2          // 场景化测试权重
}
```

**注意**：
- 权重是相对值，不是百分比
- 高权重场景会获得更多测试流量
- 权重为0的场景不会被执行

### 启用/禁用功能

通过 `enabled` 字段控制：

```json
"thinking_mode_config": {
  "enabled": true   // 启用思考模式测试
},

"context_scenarios": {
  "enabled": false  // 禁用场景化测试
}
```

### 调整超时和等待时间

```json
"request_timeout_seconds": 180,  // 请求超时时间
"wait_time_min": 1,              // 最小等待时间
"wait_time_max": 3               // 最大等待时间
```

**建议**：
- 思考模式：180秒或更长
- 常规测试：60-120秒
- 快速验证：30-60秒

### 多环境配置

可以创建多个配置文件：

```bash
# 测试环境
cp config.basic.json config.test.json
# 替换为测试环境配置

# 生产环境
cp config.basic.json config.prod.json
# 替换为生产环境配置

# 使用时指定
locust -f locustfile.py --config config.test.json
```

## 常见问题

### Q1: 不知道填什么配置？

**解决方案**：
- **base_url**：问运维或查看部署文档
- **api_key**：问管理员或查看密钥管理系统
- **model路径**：查看模型管理界面或询问模型管理员

### Q2: 配置错误会怎样？

Locust会在启动时报错，常见的错误：
- ❌ `base_url`格式错误：确保是完整的URL（包含http://）
- ❌ `api_key`无效：检查密钥是否正确
- ❌ `model`路径不存在：检查模型名称是否正确

### Q3: 如何验证配置是否正确？

```bash
# 运行一个快速测试验证
cp config.quick.json config.json
# 替换占位符
locust -f locustfile.py --users 1 --spawn-rate 1 --run-time 10s
# 如果没有报错，配置就是正确的
```

### Q4: 配置不生效怎么办？

确保：
- 修改的是 `config.json` 而不是模板文件
- JSON格式正确（可以用在线JSON验证工具检查）
- 权重不为0（至少有一个任务权重>0）
- 保存了文件修改

### Q5: 测试超时怎么办？

检查：
- `request_timeout_seconds` 是否足够大
- 思考模式建议设置为 180s 或更长
- 网络连接是否稳定
- 模型响应时间是否正常

### Q6: 报告缺少某些章节？

确认：
- 相应功能的 `enabled` 设置为 true
- 有足够的测试数据（某些分析需要最少样本数）
- 查看日志中是否有错误信息

## 最佳实践

### 1. 配置文件管理

**不要直接修改模板文件**：
- 复制模板为 config.json 后再修改
- 保留模板文件用于后续使用

**配置文件安全**：
- ⚠️ **不要**将包含真实密钥的`config.json`提交到Git
- ✅ **应该**将`config.json`添加到`.gitignore`
- ✅ **可以**将`config.*.json`模板文件提交到Git（它们只包含占位符）

**检查.gitignore**：
```bash
# 确保包含这一行
echo "config.json" >> .gitignore
```

### 2. 根据测试目标选择配置

| 测试目标 | 推荐配置 |
|---------|---------|
| 日常监控 | config.basic.json |
| 上线前测试 | config.comprehensive.json |
| 问题排查 | 定制配置 |
| 思考模式验证 | config.thinking_modes.json |
| 场景分析 | config.scenarios.json |

### 3. 权重设置原则

- 权重是相对值，不是百分比
- 高权重场景会获得更多测试流量
- 根据实际业务流量分布设置权重
- 测试重点场景设置较高权重

### 4. 超时配置建议

| 测试类型 | 建议超时 | 说明 |
|---------|---------|------|
| 思考模式 | 180s+ | 思考时间较长 |
| 常规测试 | 60-120s | 平衡速度和稳定性 |
| 快速验证 | 30-60s | 快速反馈 |

### 5. 日志和报告管理

- 定期清理旧的日志和报告
- 保持 `log_dir` 和 `report_dir` 目录整洁
- 为不同测试创建不同的报告目录

### 6. 常见使用场景

**场景1：日常性能监控**
```bash
cp config.basic.json config.json
./run.sh
```

**场景2：上线前全面测试**
```bash
cp config.comprehensive.json config.json
# 修改 base_url 和 api_key
locust -f locustfile.py --users 100 --run-time 30m
```

**场景3：思考模型优化验证**
```bash
cp config.thinking_modes.json config.json
# 运行测试后查看报告中的"思考模式性能分析"章节
```

**场景4：问题排查（只测试特定场景）**
创建自定义配置 `config.debug.json`：
```json
{
  "traffic_ratio": {
    "scenario_long_complex": 1  // 只测试长上下文复杂问题
  },
  "context_scenarios": {
    "enabled": true
  }
}
```
